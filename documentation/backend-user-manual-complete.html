1. Introduction

System Overview

The Backend Administration System is a comprehensive React-based web application designed to manage organizational content, users, and digital assets efficiently. This manual provides detailed instructions for all user roles.

🔐 Role Management

Control user permissions and access levels

👥 User Management

Manage system users and their profiles

📁 Directory System

Organize departments and staff hierarchically

📝 Content Management

Create and manage articles, banners, and media

2. Getting Started

System Access

Access the system through your web browser at:

https://your-domain.com/backend

Browser Requirements

  • Chrome 90+ (Recommended)
  • Firefox 88+
  • Safari 14+
  • Edge 90+

5. Administration Modules

5.1 Role Management

Access: System Administrators Only

Features:

  • Create and edit roles
  • Assign permissions
  • Manage access control

Creating a Role:

  1. Click + New
  2. Enter role name
  3. Select permissions
  4. Click Submit

5.2 User Management

Access: Administrators and Managers

User Information:

  • Name and email
  • Department and position
  • Assigned roles
  • Account status

Adding a New User:

  1. Click + New
  2. Fill in user details
  3. Assign role(s)
  4. Set initial password
  5. Click Submit

5.3 Directory Management

Access: Administrators and Department Managers

Directory Types:

  • 📁 Folder: Departments/Units
  • 👤 Spreadsheet: Staff Members

Creating a Department:

  1. Navigate to parent location
  2. Click + New
  3. Select "Department" type
  4. Enter department name
  5. Click Submit

Adding Staff:

  1. Navigate to department
  2. Click + New
  3. Select "Staff" type
  4. Fill in details:
    • Upload photo
    • Full name
    • Job title
    • Email and phone
    • Social media links
    • Office address
  5. Click Submit

Ordering Items:

Use Up/Down buttons to reorder items in the list

5.4 Article Management

Access: Content Managers and Administrators

Article Features:

  • Rich text editor
  • Featured images
  • Categories
  • Publication status
  • Scheduling

Creating an Article:

  1. Navigate to Articles
  2. Click + New
  3. Enter title and content
  4. Upload featured image
  5. Set publication date
  6. Click Publish

5.5 Banner Management

Access: Content Managers and Administrators

Banner Properties:

  • Banner image (1920x600px recommended)
  • Title and description
  • Link URL
  • Display order
  • Active status

Creating a Banner:

  1. Click + New
  2. Upload banner image
  3. Enter details
  4. Set link URL
  5. Click Submit
⚠️ Image Guidelines: Use optimized images under 500KB for best performance.

5.6 Programme Management

5.7 Video Management

5.8 Asset Management

5.9 VOD Management

6. Common Operations

CRUD Operations

All modules follow a standard pattern for data management:

Operation Button Description
Create + New Add new item
Read 👁 View View item details
Update ✏ Edit Modify existing item
Delete 🗑 Delete Remove item permanently

Data Tables

Data is displayed in tables with the following features:

  • Sorting: Click column headers to sort
  • Search: Use search box to filter results
  • Pagination: Navigate through pages
  • Actions: Quick access buttons for each row

Forms and Modals

Create and edit operations open in modal windows:

  1. Fill in required fields (marked with *)
  2. Upload files if needed
  3. Review entered data
  4. Click Submit or Cancel
💡 Tip: Required fields must be completed before submitting. Form validation will highlight any errors.

File Uploads

When uploading files:

  • Click "Choose File" or drag-and-drop
  • Check file size limits
  • Supported formats: JPG, PNG, GIF, PDF, DOCX
  • Wait for upload completion

Pagination

Navigate large datasets using pagination controls:

  • First: Jump to first page
  • Previous: Go back one page
  • Numbers: Jump to specific page
  • Next: Advance one page
  • Last: Jump to last page
