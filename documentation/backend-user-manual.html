1. Introduction

Welcome to the Backend Administration System

This comprehensive administration system is designed to manage all aspects of your organization's content, users, and digital assets. Built with modern web technologies, it provides a user-friendly interface for efficient content management.

System Overview

The backend administration system is a React-based Single Page Application (SPA) that communicates with a Laravel API backend. It features:

🔐 Secure Authentication

JWT-based authentication with role-based access control

📊 Modular Design

Organized modules for different content types

🎨 Responsive UI

Works seamlessly on desktop, tablet, and mobile devices

⚡ Real-time Updates

Instant data refresh and validation

Who Should Use This Manual?

  • System Administrators: Full access to all modules and settings
  • Content Managers: Create and manage articles, banners, and media
  • Department Managers: Manage directory structures and staff information
  • Regular Users: View-only access to certain modules
📘 Note: The features and modules you can access depend on your assigned role. If you need additional permissions, contact your system administrator.

2. Getting Started

System Requirements

Component Requirement
Web Browser Google Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
Internet Connection Stable broadband connection (minimum 5 Mbps)
Screen Resolution Minimum 1024x768 pixels (1920x1080 recommended)
JavaScript Must be enabled in browser

Accessing the System

The backend administration system is accessed through your web browser:

https://your-domain.com/backend
⚠️ Important: Always use HTTPS for secure access. Never share your login credentials with anyone.

First Time Setup

  1. Receive Credentials: Your system administrator will provide your username and initial password
  2. First Login: Access the login page and enter your credentials
  3. Change Password: It's recommended to change your password on first login
  4. Update Profile: Complete your profile information in the Account section
  5. Explore Features: Familiarize yourself with available modules based on your role

3. Authentication

3.1 Sign In Process

Follow these steps to log into the system:

  1. 1Navigate to the login page
  2. 2Enter your email address
  3. 3Enter your password
  4. 4Click the Sign In button
[Screenshot: Login Page Interface]

3.2 Password Requirements

For security, passwords must meet the following criteria:

  • Minimum 8 characters in length
  • At least one uppercase letter
  • At least one lowercase letter
  • At least one number
  • At least one special character (!@#$%^&*)

3.3 Password Recovery

If you forget your password:

  1. Click on "Forgot Password?" link on the login page
  2. Enter your registered email address
  3. Check your email for the password reset link
  4. Click the link and create a new password
  5. Log in with your new password
💡 Tip: Password reset links expire after 60 minutes. If your link expires, request a new one.

3.4 Session Management

The system uses token-based authentication with the following features:

  • Auto-logout: Sessions expire after 2 hours of inactivity
  • Token Refresh: Active sessions are automatically extended
  • Unauthorized Access: Invalid tokens redirect to login page

3.5 Sign Out

To safely log out of the system:

  1. Click on your profile icon in the top navigation bar
  2. Select "Sign Out" from the dropdown menu
  3. Confirm your logout if prompted
⚠️ Security Best Practice: Always sign out when using shared or public computers.

4. System Navigation

4.1 Layout Overview

The administration interface consists of several key areas:

Area Location Purpose
Top Navigation Bar Top of page Logo, user account, quick actions, sign out
Breadcrumb Menu Below navigation Shows current location and navigation path
Main Content Area Center Primary workspace for data and forms

4.2 Top Navigation Bar

The top navigation bar provides quick access to:

  • Home Icon: Returns to dashboard
  • Logo: Organization branding
  • Account Menu: Profile settings and account management
  • Sign Out: Secure logout

4.3 Breadcrumb Navigation

Breadcrumbs show your current location in the system hierarchy:

🏠 Home > Administration > Directory Management > IT Department

Click any breadcrumb link to navigate back to that level.

4.4 Dashboard

After logging in, you'll see the dashboard tailored to your role:

  • System Dashboard: Overview of system health and statistics
  • Admin Dashboard: User activities and content summaries
  • Manager Dashboard: Department-specific metrics
  • User Dashboard: Personal tasks and notifications

4.5 Maintenance Mode

During system maintenance, you may see a maintenance page. The system will be unavailable until maintenance is complete. Contact your IT department for estimated downtime.

📌 Note: Maintenance mode is controlled through environment configuration and affects all users except system administrators.

5. Administration Modules

The system is organized into specialized modules for managing different types of content and data.

5.1 Role Management

Access Level: System Administrators Only

Purpose: Define and manage user roles and permissions.

Key Features:

  • Create custom roles with specific permissions
  • Assign multiple permissions to each role
  • View all existing roles and their permissions
  • Edit role names and permission sets
  • Delete unused roles

Common Operations:

Operation Button/Action Description
Create Role + New Opens modal to create new role
Edit Role ✏ Edit Modify role details and permissions
Delete Role 🗑 Delete Remove role (requires confirmation)
⚠️ Warning: Deleting a role will affect all users assigned to that role. Ensure users are reassigned before deletion.

5.2 User Management

Access Level: Administrators and Managers

Purpose: Manage system users and their access rights.

User Information Managed:

  • Full name and email address
  • Department and position
  • Assigned roles and permissions
  • Account status (active/inactive)
  • Last login information

Creating a New User:

  1. Click + New button
  2. Fill in user details:
    • Name
    • Email address (will be used for login)
    • Department
    • Position/Title
  3. Assign role(s) to the user
  4. Set initial password (user should change on first login)
  5. Click Submit

Editing User Information:

  1. Locate user in the user table
  2. Click ✏ Edit button
  3. Update necessary fields
  4. Click Update
💡 Best Practice: Regularly review user accounts and deactivate accounts for users who no longer need access.

5.3 Directory Management

Access Level: Administrators and Department Managers

Purpose: Manage organizational structure and staff directory.

Directory Types:

  • 📁 Folder (Department): Represents departments or organizational units
  • 👤 Spreadsheet (Staff): Represents individual staff members

Hierarchical Structure:

The directory system uses a parent-child relationship:

Root
├── IT Department (Folder)
│ ├── Development Team (Folder)
│ │ ├── John Doe (Staff)
│ │ └── Jane Smith (Staff)
│ └── Support Team (Folder)
└── HR Department (Folder)

Creating a Department (Folder):

  1. Navigate to parent location
  2. Click + New
  3. Select Department type
  4. Enter department name
  5. Click Submit

Adding Staff Information:

  1. Navigate to the department
  2. Click + New
  3. Select Staff type
  4. Fill in staff details:
    • Upload photo
    • Full name
    • Job title/Occupation
    • Email address
    • Phone number
    • Social media links (Facebook, Twitter, Instagram)
    • Office address
  5. Click Submit

Ordering Items:

Control the display order of departments and staff:

  • Click ⬆ Up to move item higher in list
  • Click ⬇ Down to move item lower in list
✓ Tip: Use the breadcrumb navigation to quickly jump between department levels.

5.4 Article Management

Access Level: Content Managers and Administrators

Purpose: Create and manage website articles and blog posts.

Article Components:

  • Article categories (hierarchical)
  • Article content with rich text editor
  • Featured images
  • Publication status
  • Publication date

Creating an Article:

  1. Navigate to Articles section
  2. Select parent category (if applicable)
  3. Click + New
  4. Enter article information:
    • Title
    • Content (using rich text editor)
    • Upload featured image
    • Set publication status
    • Select publication date
  5. Preview article (optional)
  6. Click Publish

Rich Text Editor Features:

  • Bold, Italic, Underline: Text formatting
  • Headers: H1, H2, H3, etc.
  • Lists: Bulleted and numbered lists
  • Links: Insert hyperlinks
  • Images: Embed images in content
  • Code: Code blocks for technical content
📝 Note: Articles can be saved as drafts and published later. Use draft status for work-in-progress content.

5.5 Banner Management

Access Level: Content Managers and Administrators

Purpose: Manage promotional banners and sliders on the website.

Banner Properties:

  • Banner image (recommended size: 1920x600px)
  • Title and description
  • Link URL (destination when clicked)
  • Display order
  • Active/inactive status

Creating a Banner:

  1. Click + New
  2. Upload banner image
  3. Enter banner details
  4. Set link URL (optional)
  5. Set display status
  6. Click Submit
